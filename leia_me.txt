=======================================================
 Sistema de Monitoramento de Logs
 Disciplina: Técnicas de Programação 2 — CIC0198 — UnB
=======================================================

COMO COMPILAR
-------------
Pré-requisitos (Linux/WSL):
  sudo apt install g++ libgtest-dev gcovr cppcheck valgrind doxygen

Compilar os testes:
  make

Executar os testes:
  make test

Verificar cobertura de código:
  make coverage
  (Abre coverage.html no navegador)

Análise estática:
  make cppcheck

Análise dinâmica:
  make valgrind

Gerar documentação:
  make doxygen
  (Abre html/index.html no navegador)

Verificar estilo de código:
  make lint

Limpar arquivos gerados:
  make clean

ESTRUTURA DO PROJETO
--------------------
  monitora_logs.hpp         — Interface pública do módulo
  monitora_logs.cpp         — Implementação
  testa_monitora_logs.cpp   — Testes (Google Test)
  Makefile                  — Automação de build e análise
  leia_me.txt               — Este arquivo

METODOLOGIA
-----------
  Desenvolvimento orientado a testes (TDD) estrito.
  Ciclo: RED → GREEN → REFACTOR por teste.
  Mínimo de 3 commits por teste.
  Tabela de Decisão: 10 colunas (T1–T10).
  Expressões Regulares: 10 testes de caixa aberta (RE1–RE10).
